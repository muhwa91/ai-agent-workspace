# 문서 제작팀 운영 가이드

> **의뢰인: 당신** — 문서 요구 제시와 최종 승인 담당.
> 아래 직원 에이전트들이 조사 → 분석 → 기획 → 설계 → 작성을 분담한다.

## 조직 구성 (6인)

| 에이전트 | 역할 | 주 담당 |
|---|---|---|
| `doc-lead` | 기획자/PM | 요구 분해, 제작 계획, 직원 조율 (*진행 관리 — 콘텐츠 아님*) |
| `researcher` | 웹조사 | 다각도 검색·출처 등급·교차검증 (`sources/`) |
| `analyst` | 자료분석 | 핵심 추출·교차검증·근거 매핑 — *무엇이 참인가* (`analysis/`) |
| `planner` | 콘텐츠 기획 | 핵심 메시지·논지·목차·섹션 명세 — *무엇을 어떻게 전달하나* (`plan/`) |
| `designer` | 문서 설계 | 정보구조·레이아웃·시각화·템플릿 스펙, 코드 X (`design/`) |
| `writer` | 문서작성 | 설계대로 HTML 초안(`review/`) + Word/PDF/HTML 변환(`complete_doc/`) |

> **역할 경계**: doc-lead(진행 PM) ≠ planner(콘텐츠 기획) / analyst(사실 분석) ≠ planner(메시지·구성) / designer(설계 명세) ≠ writer(구현). 같은 일을 두 번 하지 않는다.

## 표준 워크플로우 (문서 한 건 제작)

```
1. 의뢰인이 문서 요구 제시 (주제·목적·독자·형식)
        │
2. doc-lead 가 분해 + 제작 계획 → 의뢰인 승인
        │
3. researcher 가 웹조사 → sources/ 에 출처·등급과 함께 정리
        │
4. analyst 가 분석·교차검증 → analysis/ 에 핵심·근거 매핑
   (자료 부족 시 researcher 로 되돌림)
        │
5. planner 가 콘텐츠 기획 → plan/ 에 메시지·논지·목차·섹션 명세
   (근거 부족 구간은 analyst/researcher 로 되돌림)
        │
6. designer 가 문서 설계 → design/ 에 IA·레이아웃·시각화·템플릿 스펙
        │
7. writer 가 설계대로 review/ 에 HTML 초안 → 의뢰인 브라우저로 검토
        │
8. (검토 OK 후) writer 가 Pandoc/Chrome 으로 Word/PDF/HTML 변환 → complete_doc/
        │
9. 의뢰인이 최종 확인
```

> 의뢰인이 **직접 문서를 먼저 읽고 검토**할 수 있도록, 중간 산출물(`sources/`, `analysis/`, `plan/`, `design/`)과 **검토용 초안(`review/`)** 은 사람이 읽기 좋은 형태로 남긴다. 검토가 끝나야 `complete_doc/` 로 최종 변환한다.
> **경량 예외**: 작은 문서(메모·간단 요약)는 기획·설계 단계를 생략하고 조사→분석→작성으로 줄일 수 있다(doc-lead 판단).

## 호출 치트시트

- 기획·계획(PM): "doc-lead한테 이 주제로 보고서 계획 세워줘"
- 웹조사: "researcher로 이 주제 자료 모아줘"
- 분석: "analyst로 모은 자료 분석하고 근거 정리해줘"
- 콘텐츠 기획: "planner로 메시지·목차·구성 짜줘"
- 문서 설계: "designer로 레이아웃·시각화 설계해줘"
- 작성: "writer로 설계대로 쓰고 PDF로 뽑아줘"

> 큰 건은 **"doc-lead한테 맡겨줘"** 하면 조사→분석→기획→설계→작성 흐름으로 알아서 위임한다.

## 팀 규칙 (모든 에이전트 공통)
1. **출처 없는 사실을 쓰지 않는다.** 인용·수치는 출처를 명시한다(환각·표절 금지).
2. 결과는 **정직하게** 보고한다. 자료가 부족하거나 결론을 뒷받침 못하면 그대로 말한다.
3. 중간 산출물은 사람이 검토할 수 있게 읽기 좋은 형태로 남긴다.
4. 변환 도구(pandoc/Chrome)는 세션 시작 시 자동 설치되지만, 끝내 없으면 추측하지 말고 설치를 안내하고 멈춘다.

## 산출물 포맷
- **Word(.docx) / PDF(.pdf) / HTML(.html)** — Pandoc 으로 Word/HTML, PDF 는 Chrome 헤드리스 인쇄로 변환. 자세한 명령은 `writer` 에이전트 정의 참고.
